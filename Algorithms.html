Algorithms Practice Questions

What is the main difference between linear search and binary search?

▼

Linear Search: Checks each element sequentially from start to end. Works on both sorted and unsorted arrays. Time complexity: O(n).

Binary Search: Repeatedly divides the search interval in half. Requires the array to be sorted. Time complexity: O(log n).

Binary search is significantly faster for large datasets but requires sorted data, while linear search works on any dataset but is slower.

Explain time complexity O(n log n) with an example.

▼

O(n log n) time complexity means the running time increases proportionally to n multiplied by the logarithm of n.

Example: Merge Sort and Heap Sort algorithms have O(n log n) time complexity.

  • Merge Sort divides the array into halves (log n divisions) and merges them back (n operations per level)
  • For an array of 1,024 elements: n log n = 1,024 × 10 = 10,240 operations (approximately)

This complexity is considered efficient and is often the best achievable for comparison-based sorting algorithms.

What is the difference between greedy algorithms and dynamic programming?

▼

Greedy Algorithms:

  • Make the locally optimal choice at each step
  • Do not reconsider previous choices
  • Faster but may not produce optimal solution for all problems
  • Examples: Dijkstra's algorithm, Huffman coding

Dynamic Programming:

  • Solves problems by combining solutions to overlapping subproblems
  • Stores results of subproblems to avoid recomputation
  • Guarantees optimal solution for problems with optimal substructure
  • Examples: Fibonacci sequence, Knapsack problem

Why is quicksort considered efficient despite having O(n²) worst-case time complexity?

▼

QuickSort is considered efficient for several reasons:

  • Average case performance: O(n log n) in practice
  • In-place sorting: Requires only O(log n) additional memory
  • Cache efficiency: Good locality of reference
  • Practical performance: Often faster than other O(n log n) algorithms due to smaller constant factors

The worst-case O(n²) occurs only when the pivot selection is poor (e.g., already sorted data with first/last element as pivot). This can be mitigated by:

  • Randomized pivot selection
  • Median-of-three pivot selection
  • Hybrid approaches (switch to heapsort for small subarrays)

What are the key characteristics of a good algorithm?

▼

A good algorithm should possess these characteristics:

  • Correctness: Produces the correct output for all valid inputs
  • Efficiency: Has reasonable time and space complexity
  • Clarity: Easy to understand and implement
  • Robustness: Handles edge cases and invalid inputs gracefully
  • Scalability: Performs well as input size increases
  • Optimality: Achieves the best possible complexity for the problem when applicable

Additionally, practical considerations include:

  • Memory usage and cache efficiency
  • Parallelization potential
  • Adaptability to different data distributions

Sorting Algorithms

Bubble Sort

Repeatedly swaps adjacent elements if they are in the wrong order.

  • O(n²)

Selection Sort

Selects the minimum element and places it at the correct position.

  • O(n²)

Insertion Sort

Builds the sorted list one element at a time.

  • O(n²)

Merge Sort

Divide-and-conquer algorithm that recursively sorts subarrays.

  • O(n log n)

Quick Sort

Uses a pivot element to partition the array.

  • Average O(n log n)

Algorithm Design Strategies

Divide & Conquer

Breaks a problem into smaller subproblems, solves them independently.

Greedy Method

Makes the locally optimal choice at each step.

Dynamic Programming

Solves problems by storing solutions of overlapping subproblems.

Backtracking

Explores all possible solutions recursively.
