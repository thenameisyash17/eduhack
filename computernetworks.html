Network Models & Basics

What are the key differences between OSI and TCP/IP models?

The OSI (Open Systems Interconnection) and TCP/IP models are both conceptual frameworks for understanding network protocols, but they have important differences:

  • Layers: OSI has 7 layers while TCP/IP has 4 layers.
  • Approach: OSI is a theoretical model while TCP/IP is a practical implementation.
  • Development: OSI was developed by ISO, TCP/IP by DARPA.
  • Protocol Dependency: OSI is protocol-independent, TCP/IP is protocol-dependent.
// TCP/IP Model Layers
1. Application Layer (HTTP, FTP, SMTP)
2. Transport Layer (TCP, UDP)
3. Internet Layer (IP, ICMP)
4. Network Access Layer (Ethernet, Wi-Fi)

Explain the difference between LAN, MAN, and WAN with examples.

Computer networks are classified based on their geographical coverage:

  • LAN (Local Area Network): Small geographic area like a home, office, or building. Example: Ethernet network in a university campus.
  • MAN (Metropolitan Area Network): Covers a city or metropolitan area. Example: Cable TV network or city-wide Wi-Fi.
  • WAN (Wide Area Network): Spans countries or continents. Example: The Internet, corporate networks connecting multiple cities.

Key differences include coverage area, ownership, transmission speed, and technology used.

Data Link & Network Layers

How does CRC (Cyclic Redundancy Check) work for error detection?

CRC is a powerful error-detecting code used in digital networks to detect accidental changes to raw data. Here's how it works:

  1. A predetermined divisor (called the generator polynomial) is agreed upon by sender and receiver.
  2. The sender appends a sequence of bits (CRC remainder) to the data such that the resulting bit sequence is divisible by the generator polynomial.
  3. The receiver divides the received bit sequence by the same generator polynomial.
  4. If the remainder is zero, the data is assumed to be error-free. Otherwise, an error is detected.
// Example CRC-32 polynomial
x³² + x²⁶ + x²³ + x²² + x¹⁶ + x¹² + x¹¹ + x¹⁰ + x⁸ + x⁷ + x⁵ + x⁴ + x² + x + 1

CRC can detect all single-bit errors, all double-bit errors (if the polynomial has at least three terms), and any odd number of errors.

What is subnetting and why is it important in IP networking?

Subnetting is the process of dividing a large IP network into smaller, more manageable subnetworks. It's crucial for several reasons:

  • Efficient IP Address Utilization: Reduces IP address wastage
  • Improved Network Performance: Reduces network congestion by limiting broadcast domains
  • Enhanced Security: Allows implementation of access controls between subnets
  • Simplified Management: Makes network administration easier

Subnetting works by borrowing bits from the host portion of an IP address to create a subnet mask. For example, a Class C network (192.168.1.0/24) can be divided into 4 subnets with a /26 mask, each supporting 62 hosts.

What are firewalls and how do they enhance network security?

A firewall is a network security device that monitors and controls incoming and outgoing network traffic based on predetermined security rules. Firewalls act as a barrier between a trusted internal network and untrusted external networks.

Types of Firewalls:

  • Packet-filtering firewalls: Examine packets and allow or block them based on source/destination IP, port, and protocol.
  • Stateful inspection firewalls: Monitor the state of active connections and make decisions based on context.
  • Proxy firewalls: Act as intermediaries between end users and the resources they access.
  • Next-generation firewalls (NGFW): Include additional features like intrusion prevention, deep packet inspection, and application awareness.

Firewalls enhance security by preventing unauthorized access, blocking malicious traffic, logging network activity, and enforcing security policies.
